# Grafana session — Research (2026-05-07)

## Goal

Add Grafana to the cluster, anonymous-viewer for dev, admin password
"Passw0rd" delivered via Secret, datasource provisioned at boot, dashboard
**created via the Grafana HTTP API** (so the operator can edit + save it),
and that dashboard added to the admin's favorites — also via API. Tag
`0.7.0`.

## Constraints (from spec + repo memory)

- Spec §7.3: Grafana provisioned automatically; `prometheus` datasource
  auto-provisioned; anonymous viewer enabled in dev; admin password via
  env in the dev overlay only.
- Spec §8: Kustomize only. No Helm. No third-party operators for Grafana.
- Repo memory: every Ingress declares exactly one Traefik entrypoint via
  `traefik.ingress.kubernetes.io/router.entrypoints`. Cluster Traefik has
  a `grafana` entrypoint bound to host port 3000 (already declared in

- Repo memory: `localhost` resolves to `::1`, k3s hostport DNAT is IPv4 —
  use `127.0.0.1` in verify scripts.
- Repo memory: do not auto-tag/merge — stop after green verify and wait.

## Why "API-driven" dashboard, not file provisioning

Grafana's file-based dashboard provisioner (`/etc/grafana/provisioning/
dashboards/*.yaml` + JSON files) loads dashboards as **provisioned**
objects. The UI surfaces them as read-only — every edit attempt warns
"This dashboard is provisioned and cannot be saved" unless the operator
copies it. The user explicitly wants the dashboard editable and saveable
at runtime. The right answer is to POST it through `/api/dashboards/db`
once Grafana is up; that creates a normal, mutable dashboard owned by the
running instance (persisted in Grafana's bundled SQLite under
`/var/lib/grafana`).

Trade-off accepted: the dashboard lives in Grafana's DB after first boot,
so wiping the Grafana PVC/emptyDir wipes saved edits. For dev that's
fine; the JSON in the repo is the source of truth on a fresh cluster, and
re-running the Job with `overwrite: true` is idempotent (and will *not*
clobber edits if the operator has bumped the dashboard version — Grafana
returns 412 on stale `version`, which the bootstrap Job ignores).

## Grafana API endpoints used

- `GET /api/health` — wait-loop until 200.
- `POST /api/dashboards/db` (basic-auth admin) with body
  `{"dashboard": {...}, "overwrite": true, "message": "bootstrap"}`.
  Returns `{ "uid": "movies-api", "id": N, "url": "..." }`.
- `POST /api/user/stars/dashboard/uid/movies-api` (basic-auth admin) —
  stars the dashboard for the authenticated user (admin). Idempotent;
  returns 200 even if already starred.

Reference: Grafana HTTP API docs (stable across 9.x→11.x).

## Image + run profile

- `grafana/grafana:11.3.0` (OSS image; works on amd64 + arm64). Default
  user is uid 472 with home `/var/lib/grafana`.
- Volumes: emptyDir on `/var/lib/grafana` (SQLite + plugins).
- `securityContext`: `runAsNonRoot: true`, `runAsUser: 472`,
  `runAsGroup: 472`, `fsGroup: 472`, drop ALL caps,
  `allowPrivilegeEscalation: false`, seccompProfile `RuntimeDefault`.
  Cannot set `readOnlyRootFilesystem: true` — Grafana writes to
  `/etc/grafana/grafana.ini` materialization paths and the plugins
  cache; out of scope to retrofit.

## Provisioning files

Even though the dashboard goes through the API, the **datasource** is
fine to provision via file (it isn't user-edited and being marked
provisioned is desirable — the operator can't accidentally repoint it
at a different Prometheus). Mounted via ConfigMap at
`/etc/grafana/provisioning/datasources/prometheus.yaml`.

## Dashboard JSON

UID `movies-api`, title "Movies API". Six panels driven by the metrics
shipped in 0.6.0 plus stock collectors:

1. Stat — Total requests (sum of `http_requests_total`).
2. Stat — In-flight (`http_requests_in_flight`).
3. Stat — 5xx rate (`sum(rate(http_requests_total{code=~"5.."}[5m]))`).
4. Time series — Requests/sec by templated route
   (`sum by (route) (rate(http_requests_total[1m]))`).
5. Time series — p95 latency by route
   (`histogram_quantile(0.95, sum by (le, route)
    (rate(http_request_duration_seconds_bucket[5m])))`).
6. Time series — `go_goroutines` (sanity check that the process
   collector arrived).

`schemaVersion: 39`, no template variables (keeps the JSON small and
working on every Grafana ≥ 10).

## Anonymous viewer + admin

`grafana.ini` via ConfigMap:

```ini
[security]
admin_user = admin
admin_password = $__env{GF_SECURITY_ADMIN_PASSWORD}

[auth.anonymous]
enabled = true
org_name = Main Org.
org_role = Viewer
```

Admin password injected via `envFrom: secretRef: grafana-admin`. The
Secret is generated by `secretGenerator` in the dev overlay so it lives
in the manifest tree but the value can be swapped per environment.
